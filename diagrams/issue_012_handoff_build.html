🗄️ Shared Infrastructure: Dataverse Session State

Central session state table shared by all agents. One table, multiple agents.

⚡
Power Automate Handoff Flow
Triggered by out-of-scope detection. Creates Dataverse record → generates summary → initiates Copilot Studio agent transfer. Logs every handoff for analytics.
📈 Analytics Power BI
📊
Handoff Dashboard
Real-time visibility into multi-agent handoff health:
  • 📈 Handoff volume — transfers per day/week by direction
  • 🔄 Round-trip rate — users bounced back (misroutes)
  • ✅ Resolution rate — queries resolved per agent
  • ⏱️ Time to resolve — pre vs. post-handoff
🚨
Automated Alerts
"HR→IT handoff rate spiked 40% this week" → may indicate a knowledge gap in the HR agent. Triggers review flow.
🎯
Knowledge Gap Detection
High handoff rates for specific topics signal missing content. Feed insights back to knowledge source curation.
🔑 Build Insight
Both agents share one Dataverse table for session state — context transfers seamlessly without custom integration. The handoff flow is a single Power Automate flow reused by every agent pair. Add a third agent? Just add another out-of-scope topic and point it at the same flow. The architecture scales horizontally — one shared infrastructure, unlimited agents.
