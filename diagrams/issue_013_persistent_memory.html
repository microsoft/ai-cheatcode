Persistent Agent Memory — Three Tiers
Issue #012 — Pattern: Conversation → Session → Long-Term Memory — From Chatbot to Assistant
💬 Tier 1: Conversation Memory Built-In
💬
What Was Said This Session
The current conversation's message history. Copilot Studio and M365 Copilot manage this natively — no custom work needed. Context window holds the last N turns.
⏱️ Ephemeral 🔄 Auto-managed 📏 Token-limited
Persistence increases ↓
📋 Tier 2: Session Memory Cross-Turn
📌
Facts Established During This Task
Structured facts extracted mid-conversation that should persist until the task is complete. Survives context window overflow — even if earlier messages scroll out, the facts remain.
📋 Task-scoped 🔑 Key-value pairs 🗑️ Cleared on task end
💡
Example
"The user is configuring Project Falcon." — established on turn 3, still available on turn 47 even after context window rotates.
🧠 Tier 3: Long-Term Memory Cross-Session
🏗️
Preferences, Past Decisions, Recurring Patterns
Persists across sessions and conversations. The agent knows who you are, what you prefer, and what you've done before. Stored externally in Dataverse or a similar store.
♾️ Persistent 🗄️ Dataverse-backed 🔒 User-scoped
💡
Example
"This user always wants answers in bullet points and prefers PowerShell over CLI." — learned last month, applied today.
Minutes Hours / Task Days → Months → Permanent
🗂️ Memory Types Classification
📌
Facts
Stable truths about the user or context
  • 📎 "User's project is Project Falcon"
  • 📎 "User is on the Contoso engineering team"
  • 📎 "Deployment target is Azure East US 2"
Rarely change · High confidence · Long retention
⚙️
Preferences
How the user likes things done
  • 📎 "Summarize in bullet points, not paragraphs"
  • 📎 "Skip introductions — get to the answer"
  • 📎 "Prefer PowerShell over Azure CLI"
Evolve over time · Medium confidence · Updateable
📅
Episodes
What happened in past interactions
  • 📎 "On March 12, we debugged the auth timeout"
  • 📎 "Last week, user asked about RBAC setup"
  • 📎 "User completed Falcon Phase 1 on April 3"
Historical · May expire · Summarized over time
⚠️ The Summarization Trap
Saving full conversation transcripts doesn't scale. Transcripts are noisy, token-heavy, and full of irrelevant turns. Extract structured facts, not raw text. Memory should be key-value pairs, not paragraphs. The naive approach wastes tokens and retrieves noise.
🔒 Privacy: Remember vs. Forget
What should agents remember? What should they forget? Memory stored in Dataverse inherits the tenant's data policies (GDPR, data residency). Always provide "Forget everything about me" as an explicit agent action. User control is non-negotiable.
