Custom Connector Patterns

🔀 When to Use What? Decision
Integration
Type?
🔄 Multiple agents reuse it?
→ Custom Connector — register once, use everywhere. Goverable as a platform asset.
🔧 One-off integration?
→ Plugin Action (Power Automate HTTP) — faster to build, no registration overhead.
⚠️ API unstable / undocumented?
→ Plugin Action — isolates instability behind a flow. Easier to update without versioning connector.
💡 Rule of Thumb
If a second team would benefit from the same integration, make it a connector.
🏗️ The Connector Stack Architecture
🤖
Copilot Studio Agent
Plugin action invokes the connector with natural-language parameter mapping. LLM decides when to call based on operation descriptions.
Conversational entity → connector parameter mapping
🔌
Custom Connector
Power Platform registration. Defines operations, schemas, and auth. Reusable across agents, flows, and apps in the environment.
Imported from OpenAPI spec → tested in connector editor
📄
OpenAPI Specification
The contract layer. Minimum: one GET, one POST, clear parameter descriptions. The LLM reads operation descriptions to decide when to call.
summary + description on every operation — scenario-specific, not just technical
☁️
Azure API Management
Production gateway: rate limiting, response caching, retry policies, request/response logging. Connector points to APIM, not directly to backend.
Enables monitoring and governance without changing the API
🏢
Internal API
Legacy ERP, custom CRM, proprietary compliance system — the enterprise data source that no pre-built connector covers.
Timeout budget: ~10 seconds before the user thinks it's broken
🔐 Authentication Patterns Security
🔑
API Key
Simplest authentication method. Key passed in header or query parameter.
Best for: Internal APIs with low security requirements
👤
OAuth 2.0 Auth Code
User-delegated access. User signs in and grants scoped permission. Agent sees only their data.
Best for: SaaS APIs with per-user data isolation
⚙️
OAuth 2.0 Client Credentials
Service-to-service, no user context. Application identity with full data access — agent filters by role.
Best for: Backend APIs, service accounts, batch processing
🛡️
Entra ID / Managed Identity
Microsoft-ecosystem native. Managed identity eliminates credential management. Inherits Power Platform security boundary.
Best for: Azure-hosted APIs, M365 integrations
🤔 Design Decision
Per-user auth (sees only their data) vs. service account (sees everything, agent filters). Choose based on data sensitivity and compliance requirements.
🔑 Architectural Insight
The OpenAPI spec is the contract. Make operation descriptions clear and scenario-specific — the LLM reads them to decide when to call the connector. NOT "GET /api/employees/{id}" → YES "Look up an employee's profile, team, and current project assignments by their employee ID." Custom connectors unlock the 20% of enterprise use cases where pre-built connectors don't reach.
