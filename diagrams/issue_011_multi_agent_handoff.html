Multi-Agent Handoff — Three Patterns Compared
Issue #010 — Pattern: Cold Transfer vs. Warm Transfer vs. Hot Transfer — choosing the right handoff architecture
❄️ Cold Transfer
★☆☆ Simple · Terrible UX
🤖 Agent A HR Agent
💬
Conversation
User asks: "I need a laptop for my new hire starting Monday." Agent A recognizes this is out of scope.
🚫
Scope Boundary
"I can't help with IT requests. Please contact the IT support bot."
✕
Context Lost — User Must Restart
🤖 Agent B IT Agent
🔄
Fresh Start
User repeats everything: "I have a new hire starting Monday, they need a laptop, they're in the engineering department..."
😤
User Friction
User must re-explain context, re-authenticate, re-navigate. High abandonment rate.
Implementation Simple redirect link or message. No shared state. No integration needed.
🤝 Warm Transfer
★★☆ Moderate · Good UX
🤖 Agent A HR Agent
💬
Conversation
User asks about laptop for new hire. Agent A detects out-of-scope query via fallback topic.
📋
Summary Generation
Azure OpenAI generates a 3-sentence summary: user's question, context discussed, and handoff reason.
📋 Summary Passed → "New hire starting Monday in engineering needs laptop provisioning"
Context variable transfer
🤖 Agent B IT Agent
📥
Context Injection
Greeting topic reads the handoff summary and injects it as system context. Agent B knows the situation.
✅
Seamless Continue
"I see you need a laptop for a new engineer starting Monday. I can help with that — what's their role level?"
Implementation Copilot Studio agent transfer + Power Automate summary generation. Moderate complexity, great ROI.
🔥 Hot Transfer
★★★ Complex · Seamless UX
🤖 Agent A HR Agent
💬
Conversation
User asks about laptop for new hire. Agent A continuously writes to shared memory during the conversation.
💾
Live State Sync
Every exchange written to shared Dataverse session table in real time. No summary needed at handoff — it's already there.
🗄️ Shared Memory Store — Dataverse AgentSession Table ← Both agents read/write
Continuous shared state
🤖 Agent B IT Agent
🔗
Full Context Access
Agent B reads entire conversation history from shared store. Knows everything Agent A discussed — including nuances.
⚡
Zero Friction
"I've reviewed your conversation with HR. I'll start the laptop provisioning for your new engineer — standard dev setup or custom?"
Implementation Shared Dataverse table + real-time sync + both agents instrumented. High complexity, best experience.
🔐 Trust Boundary — The Question Every Handoff Must Answer
🤔
Classification Trust
Does Agent B trust Agent A's intent classification? If Agent A says "this is an IT question," does Agent B verify — or just accept? Misrouted queries compound when trust is blind.
🔑
Auth Context
Does the user need to re-authenticate with Agent B? In regulated industries, the handoff may cross security boundaries that require fresh identity verification.
🔄
Misroute Recovery
What happens when Agent A misroutes? Agent B must detect "this is actually my domain" or "this isn't mine either" — and handle gracefully, not bounce endlessly.
🔑 Architectural Insight
Cold = simple but terrible UX. Warm = good UX, moderate complexity. Hot = seamless but architecturally complex. Most enterprise deployments should start with warm transfer — it delivers 90% of the UX benefit at 30% of the implementation cost. Graduate to hot transfer only when handoff volume justifies the shared infrastructure investment.
