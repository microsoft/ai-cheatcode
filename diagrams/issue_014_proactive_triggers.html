Proactive Agent Triggers — Three Types
Issue #014 — Pattern: Scheduled + Event-Driven + Threshold Triggers → Severity Router → Channel Routing
📡 Trigger Sources Detection
🕐
Scheduled Triggers
Runs on a fixed cadence — time-based scanning for actionable items.
  • ⏰ Daily at 8 AM — scan for overdue items
  • 📅 Weekly Monday — project health check
  • 📊 Monthly 1st — compliance status scan
⚡
Event-Driven Triggers
Fires when data changes — reacts to real-time events in connected systems.
  • 📄 Document uploaded — check compliance
  • 🔄 Status changed — update stakeholders
  • ✅ Approval completed — trigger next step
📈
Threshold Triggers
Activates when a metric crosses a boundary — early warning system.
  • 💰 Budget > 80% — alert owner
  • 📋 Overdue items > 5 — escalate
  • ⏱️ SLA > 24 hours — flag response

📨 Channel Routing Delivery
💬
Teams Message — Immediate
Critical alerts delivered as Adaptive Cards directly to the user in Teams. Action buttons: Open / Snooze / Ask Agent.
🔴 Critical → Immediate Push
📋
Daily Digest Card
Warnings batched into a single daily summary card at 8 AM. Groups by project, shows counts and quick links.
🟡 Warning → Daily Digest
🗨️
Next Conversation Mention
Low-priority items surfaced naturally when the user next opens the agent. "By the way, a new document was uploaded to Project X."
🟢 Info → Conversational
📧
Weekly Email Digest
Comprehensive weekly summary for stakeholders who prefer email. Includes trends, resolved items, and upcoming deadlines.
📊 All → Weekly Rollup
💡 The Value Inversion
Reactive agents save time when users ask. Proactive agents save time because users don't have to ask. Catching a missed deadline before it's missed, flagging a budget overrun before month-end — that's 10x the value of reactive responses.
⚠️ The Notification Problem
Too many alerts → notification fatigue → users ignore everything. Too few → users don't see value → agent gets turned off. Severity-based routing + user-controlled frequency + snooze/dismiss is the design principle that keeps the balance.
