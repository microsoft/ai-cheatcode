🎮 Agent Spotlight

The Proactive Health Check Agent

Pattern from Arc 4 — Proactive Agent Triggers series

The scenario: A project manager runs a team of 12 across three workstreams. Tasks live in SharePoint lists, deadlines in Planner, and compliance records in Dataverse. Every Monday morning, she spends 45 minutes manually checking each system to build a status picture. By the time she's done, the standup is half over and the information is already stale.

The solution: A proactive agent that runs a daily scan at 8 AM, queries all three systems, classifies findings by severity using Azure OpenAI, and delivers an Adaptive Card to Teams with action buttons. Critical items get an immediate ping. Warnings get batched into a daily digest. The PM opens Teams and the health check is already waiting.

The stack: Copilot Studio (conversational layer & plugin) + Power Automate (scheduled scan & delivery) + Azure OpenAI (alert classification) + Teams Adaptive Cards (delivery) + Dataverse (alert log & feedback tracking).

🧩 Pattern Breakdown

Building the Proactive Agent

Copilot Studio + Power Automate + Azure OpenAI + Teams Adaptive Cards + Dataverse

Proactive Agent build — five platform components

Why this matters: The conceptual framework from Issue #015 gives you the "what" and "why." This breakdown gives you the "how" — five components that connect into a working proactive agent. Each component is independently useful, but together they create an agent that watches, classifies, delivers, and learns.

End-to-End Flow

Power Automate recurrence trigger (daily, 8 AM)
  ↓
Query SharePoint lists + Planner + Dataverse for actionable items
  ↓
Azure OpenAI classifies each finding: 🔴 Critical / 🟠 Warning / 🟢 Info
  ↓
Write all findings to Dataverse ProactiveAlerts table
  ↓
🔴 Critical → immediate Teams Adaptive Card
🟠 Warning → batched into daily digest card
🟢 Info → surfaced in next Copilot Studio conversation
  ↓
User actions (Open / Snooze / Dismiss) → write back to Dataverse

Five components, in build order:

1.  Scheduled Scan Flow (Power Automate). Recurrence trigger set to daily at 8 AM in the user's local time zone. The flow queries three data sources: SharePoint for overdue tasks (where DueDate < today), Dataverse for stale records (where ModifiedOn > 7 days ago), and Planner for upcoming deadlines (due within 48 hours). All findings are collected into a single JSON array and passed to the next step. Keep the query scope narrow — one list, one table, one plan — to keep flow execution under 30 seconds.
2.  Alert Classification (Azure OpenAI). The JSON array of findings is sent to Azure OpenAI with a classification prompt: "Rate each item as critical, warning, or info based on time sensitivity and business impact." The model returns structured JSON with a severity tag per item. 🔴 Critical: "Compliance certificate expires tomorrow" → immediate Teams message. 🟠 Warning: "3 items overdue by 2+ days" → include in daily digest. 🟢 Info: "New document uploaded to Project X" → mention in next conversation. Use a structured output schema to ensure consistent classification.
3.  Teams Adaptive Card Delivery. Power Automate sends Adaptive Cards to Teams via the "Post adaptive card in a chat or channel" action. Each card includes: alert title, severity badge (colored circle emoji), a one-sentence AI-generated summary, and three action buttons: "Open in SharePoint" (deep-link to the source item), "Snooze 24h" (triggers a callback flow), and "Ask the Agent" (deep-links to the Copilot Studio agent with context). Critical alerts are sent immediately as individual cards. Warnings are batched into a single daily digest card at 8 AM.
4.  "Show My Alerts" Plugin Action (Copilot Studio). A plugin action in Copilot Studio that queries the ProactiveAlerts table in Dataverse, filtered to the current user and status = "pending" or "snoozed." When the user opens the agent, it proactively says: "You have 2 unacknowledged alerts from yesterday. Want me to walk through them?" This bridges the gap between push notifications and the conversational interface — alerts that were delivered to Teams can be discussed in depth through the agent.
5.  Snooze & Feedback Loop (Dataverse). The Dataverse table ProactiveAlerts stores: alert_id, user_id, severity, content, status (pending / acknowledged / snoozed / dismissed), and created_at. The "Snooze" button on the Adaptive Card triggers a Power Automate flow that updates the status and schedules a follow-up check in 24 hours. "Dismissed" items feed back into classification tuning: if a user dismisses 5 "info" alerts in a week, the flow automatically reduces info alert frequency for that user. This is the learning loop that prevents notification fatigue.
Key insight: The Copilot Studio agent itself is the simplest part of this build. The intelligence lives in the Power Automate scan flow (what to watch) and the Azure OpenAI classification (how urgent it is). Copilot Studio provides the conversational layer for follow-up — but the proactive value is delivered before anyone opens the chat.
⚠ Build warning: Adaptive Card actions in Teams have a 60-second timeout for Action.Execute callbacks. If your "Snooze" flow takes longer than that, the user sees an error. Keep the snooze flow lightweight: update one Dataverse row and return. Do any heavy processing (rescheduling, recalculating) in a separate async flow triggered by the Dataverse update.

🛠 Try This Now

Build a Weekly Health Check That Sends Itself

1 Pick one SharePoint list with a due date column. Tasks, issues, action items — anything with dates. You need at least a few overdue items to make the demo compelling. If the list is clean, add some test rows with past-due dates. This is your minimum viable data source.
2 Build the Power Automate scan flow. Recurrence trigger (weekly, Monday 8 AM). "Get items" from SharePoint filtered to DueDate lt '@{utcNow()}'. Compose the results into a JSON array. Add an HTTP action calling Azure OpenAI to generate a natural-language summary of the overdue items. This is your scan + classification in two steps.
3 Send an Adaptive Card to a Teams channel. Use the "Post adaptive card in a chat or channel" action. Include the AI-generated summary, a count of overdue items by severity, and two buttons: "Open List" (link to the SharePoint list) and "Ask the Agent" (deep-link to your Copilot Studio agent). Keep the card simple for the first iteration — a title, the summary, and the buttons.
4 Add a "Show my alerts" topic in your Copilot Studio agent. Create a topic triggered by "show my alerts" or "what did I miss" that calls a Power Automate flow to query the same overdue items. The agent summarizes them conversationally: "You have 3 overdue items this week. The most urgent is the compliance review, which was due Friday. Want me to open it?" Now your proactive card and reactive agent are connected.
